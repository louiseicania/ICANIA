# Guidance for Claude working in this repository

Read **[README.md](README.md)** in full before changing anything. It is the complete handover for this site and the section titled "Handover for the next assistant" is written specifically for you.

## The short version

- Static site, no backend, no framework. Pushing to `main` is the deploy (GitHub Pages, live at icania.co.uk).
- `index.html` is the single source of truth. The eight route pages (`sell.html`, `about.html`, `faq.html`, `how-it-works.html`, `shop.html`, `reviews.html`, `privacy.html`, `terms.html`) are **generated by `npm run build`**. Never hand edit them; edit `index.html` and rerun the build.
- Page titles and descriptions live only in the `META` object in `app.js`. The build reads them from there.
- `/brand` is a hidden design deck (`brand.html`, built from `brand.template.html` via `python3 build-brand.py`). It is `noindex` and absent from the sitemap on purpose. Keep it hidden.
- The shop at `/webshop` self updates daily from Vinted via a GitHub Action that commits `shop-data.json`. That file is generated; do not edit it. `git pull --rebase` before pushing in case the sync has run.

## Rules that matter

- **No em dashes, en dashes or hyphens in any copy a person reads.** Use commas, full stops, or reword. Hyphens in file names, urls and code are fine.
- The line art fairy never spins on its own axis; the scroll tilt on large section fairies is wanted and stronger on desktop.
- Tone is soft luxury: editorial, calm, lots of whitespace, nothing shouty.
- No analytics, trackers or cookies, ever. It is a promise on the Privacy page.

## Before you push

1. Edit source (`index.html` / `styles.css` / `app.js`).
2. `npm run build` (and `python3 build-brand.py` only if you changed the brand template).
3. Verify the result locally.
4. `git pull --rebase origin main`, then commit source and regenerated files together, then push.
